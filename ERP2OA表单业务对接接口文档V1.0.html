接口列表

POST /seeyon/rest/token
接口1：获取Token

接口说明

获取访问其他接口所需的认证Token，Token需要在后续接口的URL参数中传递。

请求头

Content-Type: application/json

请求参数

JSON 
{
    "userName": "dangantixing",
    "password": "fcb6f8a4-02ea-4d7c-89bd-4286fc846db5",
    "loginName": "人员工号"
}

响应示例

JSON 
{
    "id": "fcb6f8a4-02ea-4d7c-89bd-4286fc846db5"
}

其中id字段就是所需要的token。

Hutool调用示例

Java 
import cn.hutool.http.HttpRequest;
import cn.hutool.http.HttpResponse;
import cn.hutool.json.JSONObject;
import cn.hutool.json.JSONUtil;

public class TokenDemo {
    
    public static String getToken(String baseUrl, String loginName) {
        // 构建请求体
        JSONObject requestBody = new JSONObject();
        requestBody.put("userName", "dangantixing");
        requestBody.put("password", "fcb6f8a4-02ea-4d7c-89bd-4286fc846db5");
        requestBody.put("loginName", loginName);
        
        // 发送请求
        HttpResponse response = HttpRequest.post(baseUrl + "/seeyon/rest/token")
                .header("Content-Type", "application/json")
                .body(requestBody.toString())
                .execute();
        
        // 解析响应
        if (response.isOk()) {
            JSONObject jsonResponse = JSONUtil.parseObj(response.body());
            return jsonResponse.getStr("id");
        } else {
            throw new RuntimeException("获取Token失败: " + response.body());
        }
    }
    
    public static void main(String[] args) {

        String testBaseUrl = "http://127.0.0.1";
        
        try {
            String token = getToken(testBaseUrl, "员工工号");
            System.out.println("获取到的Token: " + token);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
POST /seeyon/rest/attachment?token={token}&firstSave=true
接口2：附件上传

接口说明

上传附件文件，获取附件ID，用于在发起流程时附加到表单的附件字段。

请求头

Content-Type: multipart/form-data

请求参数

URL参数：

  • token: 从接口1获取的token
  • firstSave: 固定值true

Body参数：

  • file: 文件内容（form-data类型）

Hutool调用示例

Java 
import cn.hutool.http.HttpRequest;
import cn.hutool.http.HttpResponse;
import cn.hutool.json.JSONArray;
import cn.hutool.json.JSONObject;
import cn.hutool.json.JSONUtil;

import java.io.File;
import java.util.ArrayList;
import java.util.List;

public class AttachmentDemo {
    
    /**
     * 上传单个附件
     */
    public static String uploadAttachment(String baseUrl, String token, String filePath) {
        File file = new File(filePath);
        if (!file.exists()) {
            throw new RuntimeException("文件不存在: " + filePath);
        }
        
        // 发送请求
        HttpResponse response = HttpRequest.post(baseUrl + "/seeyon/rest/attachment")
                .form("file", file)
                .form("token", token)
                .form("firstSave", "true")
                .execute();
        
        // 解析响应
        if (response.isOk()) {
            JSONObject jsonResponse = JSONUtil.parseObj(response.body());
            JSONArray atts = jsonResponse.getJSONArray("atts");
            if (atts != null && atts.size() > 0) {
                return atts.getJSONObject(0).getStr("fileUrl");
            }
        }
        throw new RuntimeException("上传附件失败: " + response.body());
    }
    
    /**
     * 批量上传附件
     */
    public static List uploadAttachments(String baseUrl, String token, List filePaths) {
        List attachmentIds = new ArrayList<>();
        for (String filePath : filePaths) {
            try {
                String attachmentId = uploadAttachment(baseUrl, token, filePath);
                attachmentIds.add(attachmentId);
                System.out.println("上传成功: " + filePath + " -> " + attachmentId);
            } catch (Exception e) {
                System.err.println("上传失败: " + filePath + " - " + e.getMessage());
            }
        }
        return attachmentIds;
    }
    
    public static void main(String[] args) {
        String baseUrl = "http://127.0.0.1";
        String token = "your_token_here";
        String filePath = "C:\\path\\to\\your\\file.txt";
        
        try {
            String attachmentId = uploadAttachment(baseUrl, token, filePath);
            System.out.println("附件ID: " + attachmentId);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
POST /seeyon/rest/bpm/process/third/send?token={token}
接口3：发起流程

接口说明

发起OA流程，支持两种表单类型：研发项目通用表单和研发项目变更表单。需要先获取Token和附件ID。

请求头

Content-Type: application/json
重要说明：
  • 涉及到单位、部门、人员、枚举都传编码做绑定
  • 附件控件则传附件ID数组
  • 枚举类型需要传入枚举编码

响应示例

JSON 
{
    "code": 0,
    "message": ""
}

code为0表示成功，非0表示失败。

        }
        return item;

注意事项

重要提醒：
  • 环境选择：测试环境和正式环境的地址不同，注意切换
  • Token有效期：Token可能有有效期限制，需要适时重新获取
  • 附件大小：注意附件上传的大小限制
  • 编码规范：单位、部门、人员都需要使用编码而非名称
  • 枚举值：严格按照提供的枚举编码填写
  • 日期格式：日期字段需按照要求的格式填写
  • 错误处理：建议在实际使用中添加完善的错误处理和日志记录
